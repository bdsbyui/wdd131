7s4e.org Site Plan

Bryant Smith

WDD 131-A1

Overview

Purpose

The purpose of 7s4e.org is to serve as a site of interest to my family. I am the father in a nuclear family of seven. This number and our testimony of eternal families is the inspiration for alphanumeric sequence 7S4E , i.e. Seven Smiths Forever . The intent to stengthen family identity and cohesion remains the purpose as proposed in the original plan for this site.

Audience

My family members as the intended audience is likewise unchanged, except our numbers have increased to eight, and the ninth is on the way!

Dynamic elements

The `nav` element for original home page suggest links to several child pages, though only the "Calendar" and "Games" pages were completed. The "Calandar" page displayed a complex travel itinerary for the summer of 2023, and it will be excluded from this project. Instead, this project will return to the "Games" page and build out the conceived, but not yet realized, "Recipes" page.

  • Games: Last year's effort resulted in the "Wordle" game, and this year's objective will be to make the game more mobile-friendly. However, to newly demonstrate some proficiency in using JavaScript for dynamic elements, I will add a new game, such as Minesweeper.
  • Recipes: To ensure that the requirement for "significant textual content" is satisfied, I will also implement the "Recipes" page. This course's recipes project in Unit 5 will clearly benefit this effort. However, for this personal project, I will substantively improve upon the Unit 5 assignment by customizing the design.

Content

Home page

The home page will consist of a header and a hero section. The header is a icon-based navigation menu exposed by a hamburger button. The hero section consists of a graphic introducing the family with the infinity symbol as a motif to reflect the forever theme. The graphic for the hero section is under development and will employ Canvas, SVG, or some combination of both. Very rough drafts of each are displayed below.

Draft SVG Hero

For the SVG, the challenge is that I have not yet discovered how to get the clip path to behave in Chrome and Edge as it does in Firefox. It appears that using a browser other than Firefox results in the image being clipped at the x and y limits of the path, but not along the path of the Bézier curve itself.

Draft Canvas Hero

As for Canvas, I cannot seem to get clipping path to work if the Bézier curve ends where it starts. If the start and endpoints are the same, it is as if the clip() method has no effect. The bigger challenge, however, is that the animation is proving to be considerably more complex than in the SVG. Un-commented out, the animateJacob() function as currently conceived causes the page to freeze.

Games Page

The games page has the play space at it center. Above the play space is the name of the active game. Icons or text to toggle between games is at the bottom of a mobile screen, or along the side on a desktop screen. Games include an improved version of the mostly-working Wordle and a to-be-developed implementation of Minesweeper, for which there are many examples online which can inform my own version.

Recipes Page

The recipes page is a new, to-be-developed implementation of Unit 5's Recipe Book with the addition of my own family's recipes.
